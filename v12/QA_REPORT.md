## 关键测试结果
- 欢迎封面不依赖接口或第三方图表库，HTML/CSS 首屏立即呈现。
- V12 为独立工程，不加载 V7—V11 的页面、样式或脚本。
- JavaScript 语法检查通过：`chart.js`、完整业务源码。
- 七个业务源码分片的 Git Blob 哈希与本地验收文件一致。
- 移动端无横向溢出。
- 五个页面均可正常切换。
- 交易对选择、市场搜索、排序、主题切换均可用。
- 手机买入/卖出按钮可打开下单抽屉。
- 单根 K 线轻点可以定位最近蜡烛并显示精确时间、OHLC、涨跌和成交量。
- K线拖动、鼠标滚轮缩放、双指缩放、MA/EMA/BOLL/VOL 开关可用。
- 无文字且无 `aria-label` 的按钮数量为 0。
- OKX 公共接口不可用时，种子行情、K线、盘口和模拟交易仍可展示与操作。
- 本地账户可保存持仓、委托、成交、手续费和权益变化。

## 预览中发现并修复的问题
1. 移动端侧边栏在关闭状态下仍有文字露在屏幕边缘。
   - 修复：关闭状态增加完整位移、不可见和禁用指针事件。
2. 大型业务脚本不适合通过单次 Contents API 上传。
   - 修复：按字符安全边界拆为七段，同源并行加载并在浏览器中组合执行。
3. 第二段业务源码上传时出现边界字符不一致。
   - 修复：使用 Git Blob SHA 对照本地原文件，覆盖后重新核验七段哈希。
4. 用户可能在脚本尚未加载完时快速点击“进入终端”。
   - 修复：加入点击排队与状态反馈，模块就绪后自动进入。

## 产品边界
V12 是接入公开行情的高保真模拟交易终端和客户展示原型，不是处理真实资金的生产级交易所。没有充值、提现、身份认证、生产级撮合、真实账本和托管钱包。

## 验收结论
达到“客户打开后立即看到完整展示封面，进入后可继续细看和操作”的当前交付目标。适合作为产品设计与前端实现能力展示，不应对外宣称为真实交易所。

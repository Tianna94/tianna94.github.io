# ATLAS X V12 客户展示版完整执行计划

## 1. 产品目标
- 面向客户/老板展示个人前端与产品能力，而非宣称为真实交易所。
- 打开即呈现，不允许长时间启动页、白屏或多版本串联加载。
- 第一眼有高级产品感；继续查看时，排版、交互、数据状态、按钮逻辑经得起细看。

## 2. 演示叙事
1. 即时欢迎封面：面向“邓总”的一句话介绍、产品能力摘要、进入终端按钮。
2. 交易主屏：行情摘要 → K线 → 盘口/成交 → 下单 → 持仓。
3. 二级页面：行情、资产、分析、智能洞察，所有入口均有完整内容。
4. 明示边界：公开行情 + 本地模拟交易，不涉及真实资金。

## 3. 七重角色审核
### 产品经理
- 删除无法形成闭环的功能。
- 只保留客户能理解、能操作、能展示能力的页面。
- 所有入口有明确目的和反馈。

### 专业交易员
- 信息顺序：交易对、最新价、涨跌、24h区间、K线、盘口、下单、持仓。
- K线轻点显示时间/OHLC/成交量；拖动、缩放、周期切换可用。
- 买卖语义、委托类型、数量、价格、手续费、可用余额准确。

### UX 设计师
- 手机端主屏不强塞桌面面板；下单使用底部抽屉。
- 触控目标最小 44px；关键操作不被浏览器底栏遮挡。
- 文本不重叠、不截断、不横向溢出。

### 视觉设计师
- Pearl Glass 白色体系，减少大面积模糊，保持清晰边界和层级。
- 统一 4/8px 间距系统、圆角、阴影、字体与颜色。
- 全部功能图标使用同一套线性 SVG，不使用杂乱 Unicode 符号。

### 前端架构师
- V12 独立运行，不依赖旧版本 DOM/CSS/JS。
- 零框架、零字体依赖；图表自绘 Canvas，避免第三方库阻塞。
- 首屏 HTML/CSS 立即呈现，实时行情异步增强。

### 性能工程师
- 首屏无需等待网络；种子行情与缓存先显示。
- Canvas 使用 requestAnimationFrame；ResizeObserver 节流。
- 行情请求失败自动使用缓存/模拟行情，不阻塞交互。

### QA
- 390x844、430x932、768x1024、1440x900 四档布局。
- 检查欢迎页、导航、交易对选择、周期、指标、K线轻点、下单、页面切换、主题、刷新恢复。
- 自动检测横向溢出、图表尺寸、按钮无响应、控制台错误。

## 4. 页面与功能清单
- 欢迎封面：客户称呼、能力概览、进入按钮、演示边界。
- 交易：现货/永续、交易对、K线、MA/EMA/BOLL/VOL、盘口、成交、模拟下单、持仓。
- 行情：主流币列表、搜索、涨跌/成交额排序。
- 资产：权益、可用余额、持仓、资产分布、导出。
- 分析：收益、回撤、胜率、风险、交易纪律。
- 智能：基于价格区间、盘口和账户风险的规则洞察，明确非投资建议。

## 5. 验收门槛
- 进入终端前欢迎封面 0 等待可见。
- 移动端无横向滚动；关键文字无重叠。
- 点击单根 K 线 100% 弹出数据面板。
- 页面内所有可见按钮均有响应。
- 网络断开仍可完整展示和操作模拟功能。
